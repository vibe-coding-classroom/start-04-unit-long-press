### 2. 作業任務部署細節

#### 任務 1：穩定且不疊加的長按連發 (Stable Press-and-Hold Lab)
*   **目標**：打造「零殘留」的控制迴圈。學會管理 `setInterval` 的回傳 ID，並在每次啟動前執行「防疊加清理」。
*   **Classroom 部署建議**：
    *   **生命週期實作**：配合 `touchstart` 與 `touchend` 事件，建立三段式流程。
    *   **防疊加保險**：學生必須在 `setInterval` 前一行寫入 `if (timer) clearInterval(timer)`。
    *   **驗證要點**：**資源管理審核**。學員是否理解了為什麼不清理計時器會導致「指令暴衝」？放開按鈕後，控制台是否還有殘留的發送日誌？

#### 任務 2：5 秒強制熔斷的安全熔絲 (5s Safety Kill-switch Lab)
*   **目標**：實作「自動停止」保險。防止使用者長時間按壓不放或手機當機導致機器人無限衝撞。
*   **Classroom 部署建議**：
    *   **超時停止實作**：在啟動長按的同時，設定一個 `setTimeout`，在 5 秒後無論是否放開都強制執行 `stopMove()`。
    *   **異常情境處理**：額外監聽 `window.blur` 或 `visibilitychange` 事件，確保切換 APP 時機器人能自動煞車。
    *   **驗證要點**：**失效安全 (Fail-safe) 審核**。超時保護是否真的生效？當學員在長按時「切換分頁」或「手機來電」，系統是否能自動切斷馬達輸出？

#### 任務 3：多按鈕並行不干擾 (Multi-Button Concurrency Lab)
*   **目標**：處理「雙手並行操控」。確保左手轉向、右手拉油門時，兩組定時器能協同運作而不互相覆蓋 ID。
*   **Classroom 部署建議**：
    *   **獨立定時器管理**：練習將不同按鈕的 Timer ID 分別存儲在物件或陣列中，而非共用單一變數。
    *   **數據合併發送**：嘗試將兩組狀態打包成單一 JSON 封包發送。
    *   **驗證要點**：多指操作下是否流暢？當其中一根手指放開時，另一個動作是否依然維持連發？
